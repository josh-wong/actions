# Sample Guide

Welcome to this sample guide! This document demonstrates various documentation features and serves as a comprehensive template for creating new documentation with proper formatting and structure.

## Getting Started

This section provides basic information to help you get started with the project.

### Prerequisites

Before you begin, make sure you have:

- A basic understanding of Markdown
- Access to the repository
- Familiarity with version control

## Key Features

This documentation system supports several key features:

- **MDX format** - Enhanced Markdown with component support
- **Auto-translation** - Automatic translation capabilities
- **Version control** - Full Git integration
- **Pull request workflow** - Collaborative editing process

## Basic Usage

Here's how to use this documentation:

1. **Create** - Write your content in MDX format
2. **Review** - Submit pull requests for review
3. **Merge** - Merge approved changes
4. **Translate** - Use auto-translation features as needed
