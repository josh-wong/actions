# GitHub Actions Repository Overview

Welcome to the GitHub Actions repository! This repository contains a collection of reusable workflows and automation scripts designed to streamline common development tasks across multiple projects.

## Available Actions

This repository provides several powerful automation workflows:

## Getting Started

### Step 1: Choose Your Workflow

Select the workflow that best fits your project needs:

<Tabs>
<TabItem value="translate" label="Translation Workflow">

For projects requiring multilingual documentation:

```yaml
name: Auto-Translate Documentation
on:
  pull_request:
    types: [closed]
    paths:
      - 'docs/en-us/**/*.md'
      - 'docs/en-us/**/*.mdx'

jobs:
  translate:
    if: github.event.pull_request.merged == true
    uses: josh-wong/actions/.github/workflows/auto-translate-markdown-claude-reusable.yml@main
    secrets:
      ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

### Step 2: Configure Repository Secrets

Each workflow requires specific secrets to be configured in your repository:

| Workflow | Required Secrets | Description |
|----------|------------------|-------------|
| Auto-Translate | `ANTHROPIC_API_KEY` | Claude AI API key for translations |
| Auto-Docs | `GITHUB_TOKEN` | Automatically provided by GitHub |
| Auto-PR | `GITHUB_TOKEN` | Automatically provided by GitHub |

**To add secrets:**
1. Go to your repository **Settings**
2. Navigate to **Secrets and variables** > **Actions**
3. Click **New repository secret**
4. Add the required secret name and value

## Workflow Details

### Auto-Translate Markdown Workflow

<CodeBlock language="yaml" title=".github/workflows/auto-translate.yml">
{`name: Auto-Translate Markdown to Japanese

on:
  pull_request:
    types: [closed]
    paths:
      - 'docs/en-us/**/*.md'
      - 'docs/en-us/**/*.mdx'

jobs:
  translate:
    if: github.event.pull_request.merged == true
    uses: josh-wong/actions/.github/workflows/auto-translate-markdown-claude-reusable.yml@main
    secrets:
      ANTHROPIC_API_KEY: \${{ secrets.ANTHROPIC_API_KEY }}`}
</CodeBlock>

**How it works:**
1. Triggers when a PR is merged with changes to English documentation
2. Scans for modified `.md` and `.mdx` files
3. Translates content using Claude Code AI
4. Updates front matter for Japanese documentation
5. Commits translated files to the repository

### Customization Options

#### Path Configuration

Customize which files trigger the workflows:

```yaml
paths:
  - 'docs/en-us/**/*.md'      # English markdown files
  - 'docs/en-us/**/*.mdx'     # English MDX files
  - 'content/**/*.md'         # Alternative content directory
  - '!docs/ja-jp/**'          # Exclude translations
  - '!docs/internal/**'       # Exclude internal docs
```

#### Branch Targeting

Control which branches trigger workflows:

```yaml
on:
  pull_request:
    types: [closed]
    branches: 
      - main
      - develop
      - release/*
```

#### Conditional Execution

Add conditions for workflow execution:

```yaml
jobs:
  translate:
    if: |
      github.event.pull_request.merged == true &&
      contains(github.event.pull_request.labels.*.name, 'documentation')
```

## Troubleshooting

### Common Issues

#### Workflow Not Triggering

**Symptoms**: Expected workflow doesn't run after PR merge

**Solutions**:
- Verify path filters match your file structure
- Check that PR was merged (not just closed)
- Ensure branch names match workflow configuration
- Confirm file extensions are included in path filters

#### Authentication Errors

**Symptoms**: Workflow fails with permission or authentication errors

**Solutions**:
- Verify required secrets are configured correctly
- Check API key validity and permissions
- Ensure `GITHUB_TOKEN` has necessary permissions
- Review repository access settings

#### Translation Quality Issues

**Symptoms**: Poor or inconsistent translations

**Solutions**:
- Review source content for clarity
- Use consistent technical terminology
- Avoid complex sentence structures
- Consider adding context or glossaries

### Debug Information

To debug workflow issues:

1. **Check workflow logs** in the Actions tab
2. **Review file paths** and naming conventions
3. **Verify secret configuration** in repository settings
4. **Test with minimal examples** before full deployment
