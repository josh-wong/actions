# 自動翻訳テストドキュメント

これは、自動翻訳ワークフローが正しく機能しているかを検証するために作成された基本的なテストドキュメントです。

## 目的

このドキュメントの目的は以下の通りです：

- 自動翻訳ワークフローをテストする
- 英語コンテンツが日本語に翻訳されることを確認する
- ワークフローがMDXファイルを適切に処理することを確認する
- フロントマターが正しく更新されることを検証する

## サンプルコンテンツ

### 導入

このドキュメントには、自動翻訳ワークフローが実行される際に日本語に翻訳されるべき簡単な英語テキストが含まれています。

### テスト対象の機能

1. **基本的なテキスト翻訳**: シンプルな文や段落
2. **リスト形式**: 番号付きリストと箇条書きリスト
3. **ヘッダー**: 異なるヘッダーレベル（H1、H2、H3）
4. **フロントマター**: ファイル上部のメタデータ
5. **MDXコンポーネント**: 基本的なMDX構文

### 技術的詳細

自動翻訳ワークフローは以下を実行する必要があります：

- `docs/en-us/`ディレクトリ内のファイルの変更を検出する
- `.md`および`.mdx`ファイルを処理する
- Claude Code AIを使用してコンテンツを翻訳する
- `docs/ja-jp/`ディレクトリに日本語版を生成する
- フォーマットと構造を保持する

### サンプル段落

以下は異なるタイプのコンテンツを含むサンプル段落です：

**シンプルな段落**: これは、日本語によく翻訳されるべき分かりやすい英語テキストを含む基本的な段落です。

**技術的な段落**: GitHub Actionsワークフローは、Claude Code AIを使用して英語から日本語へのドキュメント自動翻訳を実行し、元の書式と構造を維持しながら高品質な翻訳を保証します。

**混合コンテンツ**: この段落には「API」、「workflow」、「repository」などの技術用語と、適切な翻訳コンテキストが必要な一般的な英語フレーズの両方が含まれています。

## 期待される結果

ワークフローが実行された後、以下が確認できるはずです：

- `docs/ja-jp/test-auto-translate.mdx`に新しいファイルが作成される
- 全てのテキストコンテンツの日本語翻訳
- MDX構造とフォーマットの保持
- 日本語メタデータでのフロントマターの更新

## 結論

このテストドキュメントは、自動翻訳ワークフローのシンプルながら包括的なテストケースを提供します。ワークフローの機能性を検証するのに役立つ様々なタイプのコンテンツとフォーマットが含まれています。
