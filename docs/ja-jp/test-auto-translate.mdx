# 自動翻訳テストドキュメント

import TranslationBanner from '/src/components/_translation-ja-jp.mdx';

<TranslationBanner />

これは、自動翻訳ワークフローが正しく機能していることを確認するために作成された基本的なテストドキュメントです。翻訳プロセスを検証し、ドキュメント更新の適切な処理を確保するための包括的な例として機能します。

## 目的

このドキュメントの目的は以下の通りです：

- 自動翻訳ワークフローをテストする
- 英語のコンテンツが日本語に翻訳されることを確認する
- ワークフローがMDXファイルを適切に処理することを確認する
- フロントマターが正しく更新されることを検証する

## サンプルコンテンツ

### はじめに

このドキュメントには、自動翻訳ワークフローが実行される際に日本語に翻訳されるべきシンプルな英語テキストが含まれています。

### テストする機能

1. **基本的なテキスト翻訳**: シンプルな文章と段落
2. **リスト形式**: 番号付きリストと箇条書きリスト
3. **ヘッダー**: 異なるヘッダーレベル（H1、H2、H3）
4. **フロントマター**: ファイル上部のメタデータ
5. **MDXコンポーネント**: 基本的なMDX構文

### 技術的詳細

自動翻訳ワークフローは以下のことを行う必要があります：

- `docs/en-us/`ディレクトリ内のファイルの変更を検出する
- `.md`および`.mdx`ファイルを処理する
- Claude Code AIを使用してコンテンツを翻訳する
- `docs/ja-jp/`ディレクトリに日本語版を生成する
- 書式設定と構造を保持する

### サンプル段落

以下は、異なるタイプのコンテンツを含むサンプル段落です：

**シンプルな段落**: これは、日本語によく翻訳されるはずの直接的な英語テキストを含む基本的な段落です。

**技術的な段落**: GitHub Actionsワークフローは、Claude Code AIを使用して、ドキュメントを英語から日本語に自動的に翻訳し、高品質な翻訳を確保しながら元の書式設定と構造を維持します。

**混合コンテンツ**: この段落には、「API」、「ワークフロー」、「リポジトリ」などの技術用語と、適切な翻訳コンテキストが必要な一般的な英語フレーズの両方が含まれています。

## 期待される結果

ワークフローが実行された後、以下が確認されるはずです：

- `docs/ja-jp/test-auto-translate.mdx`に新しいファイルが作成される
- すべてのテキストコンテンツの日本語翻訳
- MDX構造と書式設定の保持
- 日本語メタデータでフロントマターが更新される

## 結論

このテストドキュメントは、自動翻訳ワークフロー用のシンプルながら包括的なテストケースを提供します。ワークフローの機能を確認するのに役立つ様々なタイプのコンテンツと書式設定が含まれています。
