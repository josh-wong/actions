# テストドキュメント

これは、自動翻訳ワークフローが正しく動作していることを確認するための基本的なテストドキュメントです。翻訳機能を徹底的にテストするために、様々なコンテンツタイプと書式設定要素が含まれています。

## 概要

このドキュメントには、Claude Code Actionの設定をテストするために簡単に翻訳できるシンプルなコンテンツが含まれています。

## 機能

自動翻訳ワークフローは以下を提供します：

- **自動翻訳** ドキュメントPRがマージされた際の自動翻訳
- **手動翻訳** `@claude translate docs`コメントによる手動翻訳
- **書式保持** すべてのMDX要素の書式保持
- **プロフェッショナルな日本語** 出力

## コード例

英語のまま残すべきシンプルなコードブロックを以下に示します：

```javascript
function hello() {
  console.log("Hello, world!");
  return "This code should stay in English";
}
```

## リスト

### 番号付きリスト
1. 最初の項目 - これは翻訳されるべきです
2. 2番目の項目 - これは翻訳されるべきです
3. 3番目の項目 - これは翻訳されるべきです

### 箇条書きリスト
- 機能A - 自動検出
- 機能B - 高品質翻訳
- 機能C - 書式保持

## リンクと参考資料

詳細については、[メインドキュメント](../README.md)をご確認ください。

ワークフローの詳細については、[GitHub Actionsドキュメント](https://docs.github.com/en/actions)をご覧ください。

## まとめ

このテストドキュメントは、自動翻訳ワークフローが以下のことを実行できることを確認するのに役立ちます：

1. 英語ドキュメントファイルの変更を検出する
2. 対応する日本語翻訳を作成する
3. すべての書式設定と構造を保持する
4. コードブロック、リンク、リストを正しく処理する

`docs/ja-jp/test-doc.mdx`にこのファイルの日本語翻訳が表示されれば、ワークフローは完璧に動作しています！
