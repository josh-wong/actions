# テストドキュメント

これは自動翻訳ワークフローが正しく動作することを確認するための基本的なテストドキュメントです。

## 概要

このドキュメントには、Claude Code Actionの設定をテストするために簡単に翻訳できるシンプルなコンテンツが含まれています。

## 機能

私たちの自動翻訳ワークフローは以下を提供します：

- ドキュメントPRがマージされた際の**自動翻訳**
- `@claude translate docs`コメントによる**手動翻訳**
- すべてのMDX要素の**フォーマット保持**
- **プロフェッショナルな日本語**出力

## コード例

以下は英語のままで保持されるべきシンプルなコードブロックです：

```javascript
function hello() {
  console.log("Hello, world!");
  return "This code should stay in English";
}
```

## リスト

### 順序付きリスト
1. 最初の項目 - これは翻訳されるべきです
2. 二番目の項目 - これは翻訳されるべきです
3. 三番目の項目 - これは翻訳されるべきです

### 順序なしリスト
- 機能A - 自動検出
- 機能B - 高品質な翻訳
- 機能C - フォーマット保持

## リンクと参考資料

詳細については、[メインドキュメント](../README.md)をご確認ください。

ワークフローの詳細については、[GitHub Actionsドキュメント](https://docs.github.com/en/actions)をご覧ください。

## まとめ

このテストドキュメントは、自動翻訳ワークフローが以下を実行できることを確認するのに役立ちます：

1. 英語ドキュメントファイルの変更を検出
2. 対応する日本語翻訳を作成
3. すべてのフォーマットと構造を保持
4. コードブロック、リンク、リストを正しく処理

`docs/ja-jp/test-doc.mdx`にこのファイルの日本語翻訳が表示されている場合、ワークフローは完璧に動作しています！
